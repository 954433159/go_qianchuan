## 1. 交叉编译部署

### 适用场景
- 本地开发环境为macOS/Windows，服务器为Linux
- 不想在服务器上安装Go开发环境
- 希望保持服务器环境最小化

### 实施步骤
1. 在本地执行交叉编译脚本：
   ```bash
   ./scripts/cross-compile-deploy.sh
   ```

2. 脚本会自动：
   - 清理旧构建文件
   - 交叉编译Go二进制文件为Linux x86_64架构
   - 上传到服务器
   - 停止旧服务并部署新服务
   - 验证部署是否成功

### 优缺点
**优点**：
- 服务器不需要Go环境
- 服务器上只需要一个二进制文件
- 部署过程自动化

**缺点**：
- 交叉编译可能有兼容性问题（如我们遇到的架构不匹配）
- 需要确保本地编译环境和服务器的兼容性

## 2. Docker容器化部署

### 适用场景
- 需要环境隔离和一致性
- 希望简化部署和扩展流程
- 未来可能需要水平扩展

### 实施步骤
1. 在本地执行Docker部署脚本：
   ```bash
   ./scripts/docker-deploy.sh
   ```

2. 脚本会自动：
   - 准备Docker相关文件
   - 上传到服务器
   - 在服务器上构建Docker镜像
   - 使用Docker Compose部署服务
   - 验证部署是否成功

### 优缺点
**优点**：
- 环境隔离，避免依赖问题
- 一致的开发和生产环境
- 易于扩展和管理
- 支持健康检查和自动重启

**缺点**：
- 服务器需要安装Docker
- 占用更多资源
- 学习曲线稍高

---

## 3. 源码上传构建部署

### 适用场景
- 服务器上有Go环境或可以安装
- 需要在服务器上直接修改和测试代码
- 依赖特定的服务器环境

### 实施步骤
1. 在本地执行源码构建脚本：
   ```bash
   ./scripts/source-build-deploy.sh
   ```

2. 脚本会自动：
   - 准备源码
   - 上传到服务器
   - 在服务器上安装Go环境（如果未安装）
   - 在服务器上构建应用
   - 部署并验证

### 优缺点
**优点**：
- 编译环境与运行环境一致，避免架构问题
- 可以在服务器上直接调试
- 可以使用服务器特定的环境变量或依赖

**缺点**：
- 服务器需要Go环境
- 构建过程占用服务器资源
- 需要确保服务器可以访问Go依赖下载地址

## 常见问题与解决方案

### 1. 架构不匹配问题
- **现象**：二进制文件无法在服务器上执行
- **解决方案**：确保使用正确的目标架构进行交叉编译（如GOOS=linux GOARCH=amd64）

### 2. 网络连接问题
- **现象**：无法下载Go依赖或Docker镜像
- **解决方案**：配置代理或使用国内镜像源

### 3. 权限问题
- **现象**：无法绑定80/8080等端口
- **解决方案**：使用sudo运行或使用非特权端口

### 4. 服务冲突
- **现象**：端口已被占用
- **解决方案**：停止旧服务或使用不同端口

## 推荐方案

对于千川SDK管理平台，我们推荐：

1. **开发环境**：使用Docker容器化方案，保持环境一致性
2. **生产环境**：使用交叉编译方案，最小化服务器资源占用
3. **紧急修复**：使用源码上传构建方案，快速在服务器上调试和修复

---

## 维护建议

1. **定期备份**：定期备份后端配置文件和前端构建文件
2. **监控日志**：定期检查应用日志，发现问题及时处理
3. **版本管理**：为每个部署版本打标签，便于回滚
4. **自动化**：根据项目规模，考虑集成CI/CD流程

## 回滚方案

如果新部署出现问题，可以按以下步骤回滚：

1. **交叉编译/Docker方案**：
   ```bash
   # 交叉编译回滚
   ssh root@1.12.234.253 "systemctl stop qianchuan-backend"
   ssh root@1.12.234.253 "cp /opt/qianchuan/qianchuan-server.backup /opt/qianchuan/qianchuan-server"
   ssh root@1.12.234.253 "systemctl start qianchuan-backend"
   
   # Docker回滚
   ssh root@1.12.234.253 "cd /opt/qianchuan-docker && docker-compose -f docker-compose.prod.yml down && docker-compose -f docker-compose.prod.yml up -d"
   ```

2. **源码构建方案**：
   ```bash
   # 使用Git回滚到上一个标签
   ssh root@1.12.234.253 "cd /opt/qianchuan-source && git checkout previous-tag && ./deploy.sh"
   ```
