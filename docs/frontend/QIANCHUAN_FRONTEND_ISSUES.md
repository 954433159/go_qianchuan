# 前端问题清单（功能/UX/性能/工程）

优先级分组: P0 阻断/严重 | P1 重要 | P2 一般 | P3 优化

P0/P1（需优先处理）
- 报表模块“基础版”之外的大量静态页能力未对齐（report-*.html），入口存在但功能缺失（P1）
- 全域推广仅有骨架与提示（P1），应先打通最小链路（列表/详情/创建/更新/状态）
- 路由 404 与越权场景统一体验不足：全部重定向到 dashboard，缺少明确的不可达/无权限界面（P1）

P2（中期完善）
- 账号域导航分散：安全中心/操作日志等入口分布在多处（P2）
- 创意与素材相关“审核建议/词包/关系分析”流程在部分页面有入口但未闭环（P2）
- 表单与批量操作的校验与结果提示（Toast）风格不完全统一（P2）

P3（持续优化）
- 列表大数据量时的滚动/虚拟化统一方案（已有 VirtualDataTable，可推广）（P3）
- 图表/指标卡片骨架（Skeleton）与空态（EmptyState）体验一致性（P3）
- 组件库 API 文档与 Story 演示欠缺（P3）

工程/质量问题
- 前端测试目前以 API client 与 UI 基础组件为主，缺少页面/路由/Store 的集成测试
- Sentry 面包屑已接入（API 重试记录），但未在关键业务流普及
- 国际化/i18n 未落地；可读性与可配置性可提升

对应改进详见: docs/frontend/QIANCHUAN_FRONTEND_OPTIMIZATIONS.md
