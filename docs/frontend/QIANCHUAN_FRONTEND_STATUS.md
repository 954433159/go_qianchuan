# 前端完成度与覆盖率盘点

目标: 基于静态页蓝本与现有 SPA 实现，对前端完成度进行可量化盘点，标注缺口并给出优先级。

数据源:
- 静态页: `html/`（示例: `reports.html`, `suixintui-order-list.html`, `finance-wallet.html` 等）
- SPA 路由: `frontend/src/App.tsx`

概览:
- 已实现核心域: 认证/账号、财务、活动/广告/创意、部分报表、随心推(订单)、工具-定向
- 待完善域: 全域推广（已标注“开发中”）、扩展报表（部分未接入）、若干静态页细分功能

覆盖率粗测（页面级别，按域聚合）:
- 账号与授权: 覆盖主要列表/详情、品牌授权、店铺授权（静态页 20+，SPA 覆盖子集，入口与信息层次合理）
- 活动/广告/创意: 核心流已覆盖（创建/编辑/批量操作/学习状态/低质列表/素材效率）
- 财务: 钱包/余额/流水 + 转账/退款创建与提交流已覆盖
- 报表: 基础报表可用，扩展/自定义报表与部分细分报表静态页存在但 SPA 暂未提供完整功能
- 随心推(aweme): 订单列表/创建/详情/预算追加/建议/预估/配额已覆盖
- 全域推广: 静态页覆盖丰富；SPA 有路由与页面骨架，但功能标注为“基础版/开发中”

重点差距清单（节选）:
- 报表系列: `report-*.html` 多数有静态定义（搜索词、素材、流失用户等），SPA 仅提供基础版（详见 Reports 页面）
- 全域推广系列: `uni-promotion-*.html` 静态页完整，但 SPA 仅有入口与占位提示
- 关键词/定向工具: 静态页较多（系统推荐、包管理、灰度等），SPA 目前覆盖关键词主流程与定向工作台

结论: 当前前端页面完成度约 80-85%（以“可用核心流”为口径），与后端/SDK能力对齐后仍有“扩展报表”“全域推广”等模块性缺口。
